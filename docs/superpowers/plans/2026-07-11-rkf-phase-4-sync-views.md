# RKF Phase 4 Sync And Obsidian Views Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a path-redacted, read-only multi-computer doctor plus deterministic Obsidian Base views, while keeping Google Drive wiki files as the only canonical database and allowing only the designated writer to generate named view artifacts.

**Architecture:** `rkf/sync.py` owns read probes, opaque writer registry checks, conflict/PDF/staleness detection, and compare-and-swap atomic writes. `rkf/views.py` renders five public-safe Bases from Markdown frontmatter; `views.preview` has no writes, while `views.generate` is restricted by the existing session and designated-writer checks. No code creates symlinks, junctions, `.obsidian` settings, or an Obsidian Sync configuration.

**Tech Stack:** Python 3 standard library (`dataclasses`, `hashlib`, `os`, `pathlib`, `tempfile`, `datetime`), existing RKF action runtime, `unittest`, Obsidian Bases YAML-like files, Markdown.

**Approved Design:** `docs/superpowers/specs/2026-07-10-rkf-1-1-closed-loop-design.md` sections 10–11 and 16–19.

## Global Constraints

- `connect.doctor` never calls `Workspace.ensure_base()`, never writes to wiki/raw, and never claims that cloud synchronization is current.
- Doctor receipts expose logical names and booleans/counts only; absolute paths, Drive account segments, device names, and private storage paths are redacted.
- Any conflict-copy pattern, writer mismatch, incompatible schema, or same source identity with divergent PDF checksums remains a blocker; code never selects a winner.
- Named-file writers require an expected checksum, use a temporary sibling plus atomic replace, and verify the post-write checksum before success.
- `views.preview` is read-only; `views.generate` runs only for an active designated writer and writes only `wiki/views/*.base` through the atomic writer.
- Obsidian remains a local client of the canonical wiki; `.obsidian/`, symlinks, junctions, and local semantic indexes are not committed or generated by RKF.
- Do not change writer assignment, move raw PDFs, repair conflicts, delete artifacts, create automations, commit, or push in this phase.

```python
def atomic_write_text(path: Path, text: str, *, expected_checksum: str | None) -> AtomicWriteResult:
    current = sha256_file(path) if path.exists() else ""
    if expected_checksum is not None and current != expected_checksum:
        return AtomicWriteResult(False, "checksum-mismatch", current, current)
    return _write_temp_then_replace(path, text)
```
