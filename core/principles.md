# RKF Principles

1. Reading state should compound across sessions.
2. Candidates can start drafts, but candidates are not claim evidence.
3. Human feedback increases maturity and trust.
4. Evidence matters at the claim, synthesis, citation, and publication
   boundary.
5. Private PDFs stay outside public Git.
6. Durable article text is not a data layer.
7. Topics prevent discovery drift.
8. Graph links make memory inspectable.
9. Lint reports and plans; it does not silently rewrite.
