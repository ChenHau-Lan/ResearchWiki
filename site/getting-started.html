BEGINNER PATH · PREVIEW FIRST

連接一個研究專案， 完成第一個 evidence loop。

RKF v1 的完整研究介面只有 Add、Ask、Read、Compare & Synthesize、Review， 外加 task-scoped Connect & Activate。先預覽每項 setup 變更，再明確 apply。

核心承諾 Paper → locator-backed Evidence → human-reviewed Claim → Synthesis
  1. 01

    CLONE

    取得中央 RKF checkout

    若使用自己的 fork，請替換 repository URL。

    git clone https://github.com/ChenHau-Lan/ResearchWiki.git
cd ResearchWiki
  2. 02

    PREVIEW → APPLY

    初始化中央 RKF

    第一行只預覽；確認內容後才執行 --apply，最後要求 strict check 回傳 ready。

    python3 tools/bootstrap_rkf.py
python3 tools/bootstrap_rkf.py --apply
python3 tools/check_install.py --strict
  3. 03

    CONNECT & ACTIVATE

    連接研究專案

    先 preview marker 與輕量 bridge，再 apply 完全相同的請求。

    python3 tools/rkf_auto_connect.py connect-project /path/to/research-project
python3 tools/rkf_auto_connect.py connect-project /path/to/research-project --apply

    每個新的 Codex task 都從 RKF OFF 開始。需要時說「啟動 RKF」，完成後說「停用 RKF」。

  4. 04

    10-MINUTE PAPER LOOP

    依序完成五條 workflow

    啟動 RKF，確認這個 project 的 connection。 workflow.add：Add 這個 DOI 作為 candidate；Promotion: none。 workflow.ask：回答這個研究問題；沒有 locator 就說證據不足。 workflow.read：記錄 p. 8, Fig. 3 的 Evidence，先保持 unreviewed。 workflow.compare-synthesize：列出 agreement、contradiction、gap 與 provisional conclusion。 workflow.review：顯示缺 locator、待確認 Evidence、disputed claims 與下一個閱讀行動。 停用 RKF。

EVIDENCE & PRIVACY BOUNDARY

開始前守住四條界線

  • Candidate metadata 與 model output 不是 stable Evidence。
  • Verified Claim 必須連到 exact-locator、human-verified Evidence。
  • PDF、article text、private path、secret、token 與 raw prompt 不進 public output。
  • Project marker 只表示 RKF available；每個 task 仍須明確 activate。

Windows 可將 python3 換成 py -3。Compatibility helpers 是 implementation detail，不會增加使用者需要理解的 workflow。
