BEGINNER PATH · PREVIEW FIRST

從 GitHub 開始， 連結第一個研究專案。

RKF 核心只需要 Git、Python 3.9+ 與 Codex app。先預覽每項變更，確認 receipt， 再建立本機資料層與專案 bridge。

三個不會發生的事 不會把 private wiki 發布上網、不會複製第二套 wiki、不會讓 RKF 永久保持啟動。
  1. 01

    CLONE

    取得 RKF

    若使用自己的 fork，請替換第一行的 repository URL。

    git clone https://github.com/ChenHau-Lan/ResearchWiki.git
cd ResearchWiki
  2. 02

    PREVIEW → APPLY

    建立本機 workspace

    第一行只預覽；確認沒有 blockers 後，第二行才建立 ignored 的本機設定並安裝 connector。

    python3 tools/bootstrap_rkf.py
python3 tools/bootstrap_rkf.py --apply --install-connector
python3 tools/check_install.py --strict
  3. 03

    CONNECT A PROJECT

    連結研究專案

    把範例路徑與名稱換成自己的專案。先 preview，再執行相同命令加上 --apply。

    python3 tools/rkf_auto_connect.py connect-project "/path/to/MyResearchProject" --project-name "MyResearchProject"
python3 tools/rkf_auto_connect.py connect-project "/path/to/MyResearchProject" --project-name "MyResearchProject" --apply
  4. 04

    ACTIVATE PER TASK

    在 Codex 開始使用

    每個新的 Codex task 都從 RKF OFF 開始。需要時明確說：

    啟動 RKF 把這段研究想法收進 RKF inbox 以明確 public-safe query 建立 paper candidate preview，先不要攝取 建立 aggregate-only dashboard preview，不要發布 把 dashboard preview render 成 private review page，不要更新 site

    乾淨安裝的 topic registry 一開始是空的；建立並審核自己的 topic 後，才改用其 search strings。

PUBLIC-SAFETY CHECK

開始前守住四條界線

  • 不要把 PDF、article text、private Drive path、token 或 API key 放進 repository。
  • RKF/ bridge 是入口，不是第二套 wiki 或 evidence store。
  • Discovery candidate 不是 evidence，也不會自動升級成 claim。
  • Dashboard 的 public-safe aggregate 仍需 exact-hash approval 才能發布。

Windows 可將 python3 換成 py -3。Shared Drive 與多電腦 writer setup 屬於進階流程，請先完成本機單機安裝。
